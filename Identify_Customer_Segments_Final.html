Step 0: Load the Data¶

There are four files associated with this project (not including this one):

  • Udacity_AZDIAS_Subset.csv: Demographics data for the general population of Germany; 891211 persons (rows) x 85 features (columns).
  • Udacity_CUSTOMERS_Subset.csv: Demographics data for customers of a mail-order company; 191652 persons (rows) x 85 features (columns).
  • Data_Dictionary.md: Detailed information file about the features in the provided datasets.
  • AZDIAS_Feature_Summary.csv: Summary of feature attributes for demographics data; 85 features (rows) x 4 columns

Each row of the demographics files represents a single person, but also includes information outside of individuals, including information about their household, building, and neighborhood. You will use this information to cluster the general population into groups with similar demographic properties. Then, you will see how the people in the customers dataset fit into those created clusters. The hope here is that certain clusters are over-represented in the customers data, as compared to the general population; those over-represented clusters will be assumed to be part of the core userbase. This information can then be used for further applications, such as targeting for a marketing campaign.

def encode_categorical_data(df, categorical_cols, method="drop"):
    """
    Encode categorical data in a DataFrame with special handling for binary categoricals.
    
    ::Parameters::
        - df: pandas DataFrame
              The input DataFrame containing the data.
        - categorical_cols: list of str
              A list of column names that contain categorical data.
        - method: str, optional (default="drop")
              The method to encode the data. Options: "drop" or "one-hot".

    ::Returns::
        - encoded_df: pandas DataFrame
              The DataFrame with encoded or dropped categorical columns.
    """

    if method not in ["drop", "one-hot"]:
        raise ValueError("Method must be 'drop' or 'one-hot'.")
        
    for col in categorical_cols:
        if col in df.columns:
            # The length of unique values is 2 so this means that it is either numeric or non-numeric
            # and these are handled as such
            if len(df[col].unique()) == 2:
                if pd.api.types.is_numeric_dtype(df[col]):
                    continue  # Binary numeric variable, no action needed
                else:
                    le = LabelEncoder()
                    df[col] = le.fit_transform(df[col])
            # If the user chose to perform one hot encoding then we need to honor that
            elif method == "one-hot":
                # For multi-level categoricals, perform one-hot encoding
                dummies = pd.get_dummies(df[col], drop_first=True)  # Use drop_first=True to drop the first category
                df = pd.concat([df, dummies], axis=1)
                df = df.drop(columns=col)
            # If the user has requested "drop" then our job is very easy
            else:
                df = df.drop(columns=col)
        else:
            print("Column {} was not found in the reduced genDemo dataframe".format(col))
            print("This is actually okay as it is possible that this column was dropped from our previous cleaning steps")
    
    
    return df

Step 2.3: Interpret Principal Components¶

Now that we have our transformed principal components, it's a nice idea to check out the weight of each variable on the first few components to see if they can be interpreted in some fashion.

As a reminder, each principal component is a unit vector that points in the direction of highest variance (after accounting for the variance captured by earlier principal components). The further a weight is from zero, the more the principal component is in the direction of the corresponding feature. If two features have large weights of the same sign (both positive or both negative), then increases in one tend expect to be associated with increases in the other. To contrast, features with different signs can be expected to show a negative correlation: increases in one variable should result in a decrease in the other.

def interpret_pca_components(pca, feature_names, n_components=3, print_all=False):
    # Check if the number of components requested is greater than the actual number of components in the PCA object
    if n_components > pca.n_components_:
        print(f"Error: The PCA object has only {pca.n_components_} components.")
        return
    
    # Loop through the desired number of components
    for i in range(n_components):
        component = pca.components_[i]
        feature_weights = zip(feature_names, component)
        
        # Sort the features by their weight in the component
        sorted_features = sorted(feature_weights, key=lambda x: x[1], reverse=True)
        
        # Display the sorted features for the i-th component
        print(f"Principal Component {i + 1} Features:")
        if print_all == True:
            for feature, weight in sorted_features:
                print(f"{feature}: {weight}")
            print("\n")
        else:

            for feature, weight in sorted_features[:5]:

            print("=============================")
            print("Bottom 5 Features and Weights")
            for feature, weight in sorted_features[-5:]:
                print(f"{feature}: {weight}")
            print("=============================")
            print("=============================")

def KMeans_ScreePlot(X_pca, max_clusters=20, RANDOM_STATE=0):
    
    upperBound = max_clusters+1
    scores = []
    centers = list(range(1,upperBound))
    
    for cluster in range(1,upperBound):
        kmeans = KMeans(n_clusters=cluster,
                        random_state=RANDOM_STATE)
        kmeans.fit(X_pca)
        scores.append(np.abs(kmeans.score(X_pca)))
    
    plt.figure(figsize=(12,6))
    plt.plot(centers, scores, linestyle='--', marker='o', color='b')
    plt.xlabel('# of Clusters')
    plt.ylabel('Sum of Square Errors (SSE)')
    plt.title('Elbow Plot: SSE vs # of Clusters')
    plt.xticks(centers)
    
    return None
